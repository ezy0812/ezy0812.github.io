overview

two key network-layer functions

forwarding

move packets from router’s input to appropriate router output

routing

determine route taken by packets from source to destination

data plane

  • determines how datagram arriving on router input port is forwarded to router output port
  • forwarding function

control plane

  • determines how datagram is routed among routers along end-end path from source host to destination host

    two approaches

  • traditional routing algorithms
    implemented in routers
  • software-defined networking
    implemented in remote servers

Router

hign-level view of route architecture

input port

Two forwarding methods

  • destination-based forwarding
    forward based only on destination IP address
  • generalized forwarding
    forward based on any set of header field values

switching fabrics

via memory

  • traditional computers with switching under direct control of CPU
  • packet copied to system’s memory
  • speed limited by memory bandwidth(2 bus crossings per datagram)

    via a bus

  • datagram from input port memory to output port memory via a shared bus
  • bus contention: switching speed limited bu bus bandwidth

    via interconnection network

  • overcome bus bandwidth limitations
  • advanced design: fragmenting datagram into fixed length cells, switch cells through the fabric

input port queueing

  • fabric slower than input ports combined leading to queueing at input queues
  • Head-of-the-Line (HOL) blocking: queued datagram at front of queue prevents others in queue from moving forward

    output ports

  • buffering when arrival rate via switch exceeds output line speed

    scheduling mechanisms

  • FIFO scheduling
  • discard policy
    • tail drop: drop arriving packet
    • priority: drop/remove on priority basis
    • random: drop/remove randomly

IP

IP datagram format

subnet

device interfaces with same subnet part of IP address, can physically reach other without intervening router

CIDR-Classless Inter Domain Routing

  • subnet portion of address of arbitrary length
  • address format: a.b.c.d/x, where x is the number of bits in subnet portion of address

    How to get IP address?

  • hard-coded by system admin in a file
  • DHCP: Dynamic Host Configuration Protocol–dynamically get address from as server

DHCP

  • goal: allow host dynamically obtain its IP address from network server when it joins network
  1. can renew its lease on address in use
  2. allows reuse of addresses (only hold address while connected “on”)
  3. support for mobile users who want to join network

client-server scenario

return more than just allocated IP address on subnet

  • address of first-hop router for client
  • name and IP address of DNS server
  • network mask

NAT–network address translation

IPv6

  • initial motivation: 32-bit address space soon to be completely allocated.
  • additional motivation:
    1. header format helps speed processing/forwarding
    2. header changes to facilitate QoS

datagram format

  • priority: identify priority among datagrams in flow
  • flow Label: identify datagrams in same “flow” (concept of “flow” not well defined).
  • next header: identify upper layer protocol for data
  • checksum: removed entirely to reduce processing time at each hop
  • options: allowed, but outside of header, indicated by “Next Header” field
  • ICMPv6: new version of ICMP
    1. additional message types, e.g. “Packet Too Big”
    2. multicast group management functions
