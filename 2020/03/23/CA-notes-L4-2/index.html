Instruction Formats

I-Format: instructions with immediates

lw/sw (offset is immediate), and beq/bne

But not the shift instructions

J-Format: j and jal

But not jr

R-Format: all other instructions

R-Format

opcode rs rt rd shamt funct
6 5 5 5 5 6
set 000 000 for all R-format instructions source register target register destination register set to 0 in all but the shift instructions specifies the instruction,can encode $2^6=64$ instructions
> add dst,src1,src2 $\rightarrow$ add rd,rs,rt

I-Format

opcode rs rt immediate
6 5 5 16
specifies the instruction source register target register 2’s complement number
> addi $21,$22,-50 $\rightarrow$ addi R[rt]=R[rs]+signExtImm
> field representation (decimal)
> opcode rs rt immediate
> :-: :-: :-: :-:
> 8 22 21 -50

dealing with large immediates

$\color{red}{lui(Load; Upper; immediate)}$
lui reg,imm

move 16-bit into upper half(bits 16-31) of reg and zeros the lower half(bits 0-15)

lui example

addi $t0, $t0, 0xababcdcd
translates into

lui $at, 0xabab
ori $at, $at, 0xcdcd
add $t0, $t0, $at

beq and bne

immediate is the number of instructions to jump


opcode|rs|rt|immediate

4|9|0|3

Questions on PC-addressing

Does the value in branch immediate field change if we move the code?

  • If moving individual lines of code, then yes
  • If moving all of code, then no

If destination is > $2^{15}$ instructions away from branch

beq $s0, $0, far
# next instru ...

$$\Downarrow$$

bne $s0, $0, next
j far
next: next instru ...

J-Format

opcode target address
6 26

We can specify $2^{26}$ addresses, and because of word-aligned, it brings us to 28 bits address

Take the 4 highest order bits from the PC

New PC={(PC+4)[31:28], target address, 00}

Branches use PC-relative addressing, Jumps use absolute addressing
