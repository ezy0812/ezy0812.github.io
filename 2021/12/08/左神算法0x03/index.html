前缀树

有一些字符串数组，对这个数组生成前缀树

节点类如下

public static class TrieNode {
    public int path;
    public int end;
    public TrieNode[] nexts;

    public TrieNode() {
        path = 0;
        end = 0;
        nexts = new TrieNode[26];
    }
}

path值指的是经过该节点的字符串有几个，end值指的是以该节点结尾的字符串有几个，大小为26的nexts数组表示26个字母，每个字母下面连着哪条路，为null当然是指没有指向这个字母的路，具体如图

根节点的含义代表的就是空字符串

前缀树具体有什么作用呢？

1）可以查询某个字符串被放进去多少次，只需要顺着nexts数组往下找，直到字符串结束，查看这个节点的end值即可，碰到null当然是没有这个字符串

2）可以查询以某个字符串为前缀的字符串有多少个，同理，顺着字符串直到结束，查看这个节点的path值即可

public static class trieTree{
    private TrieNode root;

    public trieTree(){
        root = new TrieNode();
    }

    public void insert(String str){
        if(str == null) return;
        TrieNode node = root;
        char[] chars = str.toCharArray();
        int index;
        for(char chr:chars){
            index = chr - 'a';
            if(node.nexts[index] == null) node.nexts[index] = new TrieNode();
            node.path++;
            node = node.nexts[index];
        }
        node.path++;
        node.end++;
    }

    public void delete(String str){
        if(search(str) != 0){
            TrieNode node = root;
            char[] chars = str.toCharArray();
            int index;
            node.path--;
            for(char chr:chars) {
                index = chr - 'a';
                if(--node.nexts[index].path == 0){
                    node.nexts[index] = null;
                    return;
                }
                node = node.nexts[index];
            }
            node.end--;
        }
    }

    public int search(String str){
        if(str == null) return 0;
        TrieNode node = root;
        char[] chars = str.toCharArray();
        int index;
        for(char chr:chars){
            index = chr - 'a';
            if(node.nexts[index] == null) return 0;
            node = node.nexts[index];
        }
        return node.end;
    }

    public int countPrefix(String str){
        if(str == null) return 0;
        TrieNode node = root;
        char[] chars = str.toCharArray();
        int index;
        for(char chr:chars){
            index = chr - 'a';
            if(node.nexts[index] == null) return 0;
            node = node.nexts[index];
        }
        return node.path;
    }
}

贪心算法

在某一个标准下，优先考虑最满足标准的样本，最后考虑最不满足标准的样本，最终得到一个答案的算法

解题套路（对数器验证自己的贪心算法是否正确）：

  1. 实现一个暴力绝对正确的方法X
  2. 使用对数器，去验证自己的贪心策略
  3. 不要去纠结贪心策略的证明

0x01 安排会议

一些项目要占用一个会议室宣讲，会议室不能同时容纳两个项目的宣讲。 给你每一个项目开始的时间和结束的时间(给你一个数组，里面是一个个具体的项目)，你来安排宣讲的日程，要求会议室进行的宣讲的场次最多， 返回这个最多的宣讲场次。

贪心策略：优先选择结束时间最早的

public static class Program {
    public int start;
    public int end;

    public Program(int start, int end) {
        this.start = start;
        this.end = end;
    }
}

public static class programComparator implements Comparator<Program>{
    @Override
    public int compare(Program o1, Program o2) {
        return o1.end - o2.end;
    }
}

public static int bestArrange(Program[] programs, int start){
    int result = 0;
    Arrays.sort(programs, new programComparator());
    for(Program each:programs){
        if(start<=each.start){
            start = each.end;
            result++;
        }
    }
    return result;
}

0x02 分金条

一块金条切成两半，是需要花费和长度数值一样的铜板的。比如长度为20的金条，不管切成长度多大的两半，都要花费20个铜板。

一群人想整分整块金条，怎么分最省铜板? 例如,给定数组{10,20,30}，代表一共三个人，整块金条长度为10+20+30=60。 金条要分成10,20,30三个部分。 如果先把长度60的金条分成10和50，花费60； 再把长度50的金条分成20和30，花费50；一共花费110铜板。 但是如果先把长度60的金条分成30和30，花费60；再把长度30金条分成10和20， 花费30；一共花费90铜板。

输入一个数组，返回分割的最小代价

哈弗曼树思想，把数组放到小根堆中，每次用两个最小值，然后把和再次放入小根堆中，循环即可

public static int lessMoney(int[] arr){
    PriorityQueue<Integer> pq = new PriorityQueue<>();
    for(int each:arr) pq.add(each);
    int sum = 0;
    int temp;
    while(pq.size()>=2){
        temp = pq.poll() + pq.poll();
        sum += temp;
        pq.add(temp);
    }
    return sum;
}

0x03 项目最大收益

输入：正数数组costs 正数数组profits 正数k 正数m

costs[i]表示i号项目的花费，profits[i]表示i号项目在扣除花费之后还能挣到的钱(利润)，k表示你只能串行的最多做k个项目，m表示你初始的资金

每做完一个项目，马上获得收益，可以支持你去做下一个项目

输出： 你最后获得的最大钱数

小根堆用于存放按成本排列的项目，大根堆用于存放按利润排列的项目，每次从小根堆拿出基于目前已有的资金可以做的项目，放入大根堆中，然后拿出利润最大的一个，循环往复，直到做完k个项目或者拿不出可以做的项目而提前结束

public static class Node{
    int cost;
    int profit;

    public Node(int cost, int profit) {
        this.cost = cost;
        this.profit = profit;
    }
}

public static class minCost implements Comparator<Node>{
    @Override
    public int compare(Node o1, Node o2) {
        return o1.cost - o2.cost;
    }
}

public static class maxProfit implements Comparator<Node>{
    @Override
    public int compare(Node o1, Node o2) {
        return o2.profit - o1.profit;
    }
}

public static int maxGain(int k, int m, int[] costs, int[] profits){
    Node[] nodes = new Node[costs.length];
    for(int i=0;i< costs.length;i++) nodes[i] = new Node(costs[i], profits[i]);
    PriorityQueue<Node> pqCost = new PriorityQueue<>(new minCost());
    PriorityQueue<Node> pqProfit = new PriorityQueue<>(new maxProfit());
    for(Node each:nodes) pqCost.add(each);
    for(int i=0;i<k;i++){
        while(!pqCost.isEmpty() && pqCost.peek().cost <= m) pqProfit.add(pqCost.poll());
        if(pqProfit.isEmpty()) return m;
        m += pqProfit.poll().profit;
    }
    return m;
}

0x04 （不属于贪心）另一个大根堆小根堆结合的应用

要求对给定数据流处理，随时可以取得中位数

第一个数进入大根堆，后面进来的数

1）<= 大根堆顶，进入大根堆，否则进入小根堆

2）一旦大根堆和小根堆的size之差 =2，则将个数多的堆弹出放到另一个堆

public static class medianHolder{
    private PriorityQueue<Integer> smallHeap = new PriorityQueue<>();
    private PriorityQueue<Integer> bigHeap = new PriorityQueue<>(new bigHeapComparator());

    private void modifySize(){
        int size1 = smallHeap.size();
        int size2 = bigHeap.size();
        if(Math.abs(size1 - size2) == 2){
            if(size1 > size2) bigHeap.add(smallHeap.poll());
            else smallHeap.add(bigHeap.poll());
        }
    }

    public void addNum(int num){
        if(bigHeap.isEmpty() || num <= bigHeap.peek()) bigHeap.add(num);
        else smallHeap.add(num);
        modifySize();
    }

    public Integer getMedian(){
        int size1 = smallHeap.size();
        int size2 = bigHeap.size();
        if(size1 + size2 == 0) return null;
        Integer head1 = smallHeap.peek();
        Integer head2 = bigHeap.peek();
        if((size1 + size2) % 2 == 0) return (head1 + head2) / 2;
        return size1 > size2 ? head1 : head2;
    }

}

public static class bigHeapComparator implements Comparator<Integer>{
    @Override
    public int compare(Integer o1, Integer o2) {
        return o2 - o1;
    }
}

暴力递归

1，把问题转化为规模缩小了的同类问题的子问题

2，有明确的不需要继续进行递归的条件(base case)

3，有当得到了子问题的结果之后的决策过程

4，不记录每一个子问题的解

汉诺塔

打印n层汉诺塔从最左边移动到最右边的全部过程

假设函数为P(i, from, to, other)

一共i层，从from柱子移动到to柱子，other代表其他的柱子

整个递归过程即为

1）将1 - i-1层的圆盘移动到other柱子 P(i-1, from, other, to)

2）将i层圆盘移动到to柱子 打印即可

3）将1 - i-1层的圆盘移动到to柱子 P(i-1, other, to, from)

不需要考虑中间比如第1步或者第3步移动的过程中怎么保证大圆盘没有放到小圆盘上，这个问题交给第1步或者第3步的递归过程即可

public static void hanoi(int n){
    if(n > 0) P(n, "left", "right", "mid");
}

public static void P(int i, String from, String to, String other){
    if(i == 1) {
        System.out.println("Move 1 from " + from + " to " + to);
        return;
    }
    P(i-1, from, other, to);
    System.out.println("Move " + i + " from " + from + " to " + to);
    P(i-1, other, to, from);
}

子序列

打印字符串全部子序列，包括空字符串

从头开始，选择要该字符还是不要该字符，分出两条，随后分别递归

public static void allSubSequences(String str){
    char[] chars = str.toCharArray();
    process(chars, 0);
}

public static void process(char[] chars, int i){
    if(i == chars.length){
        System.out.println(String.valueOf(chars));
        return;
    }
    process(chars, i+1);
    
    char tmp = chars[i];
    // 0用于占位，代表不要该字符（前提是原来的字符串中没有这个ASCII码为0的字符，ASCII码为0的字符为空字符）
    chars[i] = 0;
    process(chars, i+1);
    
    chars[i] = tmp;
}

全排列

打印一个字符串的全部排列，要求不要出现重复的排列

0 - i-1 位置定好字符，从i位置之后的所有字符都可以放在i位置进行尝试（注意重复）, 定好i位置的字符，就进入下一层递归

process(int i, char[] chars)

public static void allPermutations(String str){
    if (str == null || str.length() == 0) {
        return;
    }
    char[] chars = str.toCharArray();
    process(0, chars);
}

public static void swap(char[] chars, int i, int j){
    char tmp = chars[i];
    chars[i] = chars[j];
    chars[j] = tmp;
}

public static void process(int i, char[] chars){
    if(i == chars.length){
        System.out.println(String.valueOf(chars));
        return;
    }
    boolean[] visit = new boolean[26];
    for(int j = i;j < chars.length; j++){
        if(!visit[chars[j]-'a']){
            visit[chars[j]-'a'] = true;
            swap(chars, j, i);
            process(i+1, chars);
            swap(chars, j, i);
        }
    }
}

分数最高

arr=[1,2,100,4], A先拿，B后拿，双方都只能拿最左或最右的值，谁最后最大谁获胜

先手函数 f() 后手函数 s()

对于先手函数，base case: 即只剩一个数，当然返回该值

否则返回max[（拿了左边的+后手操作返回值）,（拿了右边的+后手操作返回值）]

对于后手函数，base case: 只剩一个数，既然是后手，那么该数当然不是自己拿，而是对方拿，所以返回0

否则返回min[（去掉左边值然后先手操作）,（去掉右边值然后先手操作）]

为什么是最小值？因为这是后手函数，自己是后手，那么对方一定会选择对他优势更大的，所以留给自己的当然是较小值的情况

public static int f(int[] arr, int l, int r){
    if(l == r) return arr[l];
    return Math.max(arr[l] + s(arr, l+1, r), arr[r] + s(arr, l, r-1));
}

public static int s(int[] arr, int l, int r){
    if(l == r) return 0;
    return Math.min(f(arr, l+1, r), f(arr, l, r-1));
}

public static int highestScore(int[] arr){
    if(arr == null || arr.length == 0) return 0;
    return Math.max(f(arr, 0, arr.length-1), s(arr, 0, arr.length-1));
}

递归逆序栈

不能使用额外数据结构

函数f: 拿出来栈底的值，其他值按原来顺序放好

每次弹出栈顶，递归调用直到为空，并且记录每次弹出的值，该值不是最后一个值就再压进去，把最后一个值一层一层往上返回

public static int f(Stack<Integer> stack){
    int result = stack.pop();
    if(stack.isEmpty()){
        return result;
    }else{
        int last = f(stack);
        stack.push(result);
        return last;
    }
}

public static void reverseStackUsingRecursive(Stack<Integer> stack){
    if(stack.isEmpty()) return;
    int last = f(stack);
    reverseStackUsingRecursive(stack);
    stack.push(last);
}

数字转化字母

1 - A 2-B 3-C … 26个字母皆如此，那么一个字符串”111”可以转化为”AAA”, “KA”, “AK”，则给定一个只有数字字符组成的字符串str，返回有多少种转化结果

0, …, i-1 位置定好转化目标，然后返回 i, …, length - 1的结果

检查位置i上的数字，

如果为0，那么无论如何都没法转化，整个决定错误，返回0

不为0，则一定可以将该位置转化为对应字母，返回i+1 - length-1的结果

然后分为该数字为1，2，其他

1）为1，则还可以跟下一个数字组合成一个两位数，转化为对应字母（如果存在下一个）

2）为2，并且下一个数字在0-6之间

3）其他数字，莫得办法

public static int process(char[] str, int i){
    if(i == str.length) return 1;
    if(str[i] == '0') return 0;
    int res = process(str, i+1);;
    if(str[i] == '1'){
        if(i+1 < str.length) res += process(str, i+2);
        return res;
    }
    if(str[i] == '2'){
        if((i+1 < str.length) && (str[i+1] >= '0') && (str[i+1] <= '6')) res += process(str, i+2);
        return res;
    }
    return res;
}

public static int convertToLetterString(String str){
    if(str == null || str.length() == 0) return 0;
    return process(str.toCharArray(), 0);
}

背包问题

给定两个长度都为N的数组weights和values，weights[i]和values[i]分别代表 i号物品的重量和价值。给定一个正数bag，表示一个载重bag的袋子，你装的物品不能超过这个重量。返回你能装下最多的价值是多少

依然是从头开始尝试，0, …, i-1 位置定好选择装哪个物品，并记录已装物品的重量，然后返回 i, …, length - 1的结果

public static int process(int[] weights, int[] values, int bag, int alreadyweight, int i){
    if(alreadyweight > bag) return 0;
    if(i == weights.length) return 0;
    return Math.max(values[i]+process(weights, values, bag, weights[i] + alreadyweight, i+1),
            process(weights, values, bag, alreadyweight, i+1));
}

public static int Knapsack(int[] weights, int[] values, int bag){
    if(weights == null|| values == null || weights.length != values.length || values.length == 0) return 0;
    return process(weights, values, bag, 0, 0);
}

N皇后问题

在N*N的棋盘上要摆N个皇后，要求任何两个皇后不同行、不同列， 也不在同一条斜线上

给定一个整数n，返回n皇后的摆法有多少种

n=1，返回1

n=2或3，2皇后和3皇后问题无论怎么摆都不行，返回0

n=8，返回92

1.0

对N*N棋盘进行深度优先遍历，第一行选一个位置放一个皇后，进入下一行，选一个位置放一个皇后，并且保证与前面放的皇后不重合

public static int nQueens(int n){
    if(n < 1) return 0;
    int[] records = new int[n];
    return process(0, records, n);
}

// 当前在第i行，records[m] = n 表示第m行的皇后放在第n列
public static int process(int i, int[] records, int n){
    // 0到n-1行完成放皇后的操作，说明n*n的棋盘完成摆放，返回1即为1个n皇后的摆法
    if(i == n) return 1;
    int res = 0;
    for(int k=0;k<n;k++)
        // 尝试每一列
        if(isValid(i, k, records)){
            records[i] = k;
            res += process(i+1, records, n);
        }
    return res;
}

// i行j列放皇后是否合适
public static boolean isValid(int i, int j, int[] records){
    for(int m=0;m<i;m++)
        if(records[m] == j || Math.abs(records[m] - j) == Math.abs(m - i))
            return false;
    return true;
}

2.0 位运算优化

假设为8皇后，则设立限制为前32-8位为0，后8位为1 （32是指不能超过32皇后）

存在三个数字，分别为列限制，左斜线限制，右斜线限制

从第0行，放在第几列，则把列限制的哪一位置1，然后左斜线限制的该位的左一位置1，右斜线限制的该位的右一位置1，

假设列限制为 0000 1 000，

则左斜线限制为0001 0 000，

右斜线限制为 0000 0 100

那么第1行，结合已有的三个限制（即或运算），将所有限制结合在一个数上，即为000 111 00，将该数取反然后和初始的限制 32-8 个0 8个1 做与运算，则为32-8 个0 111 000 11，所以1即为可以放皇后的位置，通过每次取得最右侧的1，然后继续递归，将三个限制做处理，然后传递到下一层，首先列限制即为将原先的列限制和取得的最右侧的1做或运算，左斜线限制为将原先的左斜线限制和取得的最右侧的1做或运算，然后左移一位，右斜线限制为将原先的右斜线限制和取得的最右侧的1做或运算，然后右移一位

public static int process2(int limit, int colLimit, int leftDiaLimit, int rightDiaLimit){
    if(limit == colLimit) return 1;
    int res = 0;
    int positions = limit & (~(colLimit|leftDiaLimit|rightDiaLimit));
    int mostRight;
    while(positions != 0){
        mostRight = positions & ((~positions)+1);
        positions = positions - mostRight;
        res += process2(limit, colLimit|mostRight, (leftDiaLimit|mostRight)<<1,
                (rightDiaLimit|mostRight)>>>1);
    }
    return res;
}

public static int nQueens2(int n){
    if(n < 1 || n > 32) return 0;
    int limit = (n == 32) ? -1 : (1<<n)-1;
    return process2(limit, 0, 0, 0);
}
